进入多版本片源时，让 LinPlayer 按你的正则偏好自动选中合适的版本、字幕轨和音频轨，不用每次手动挑。

<!-- more -->

## 在哪里设置

- **移动端 / 桌面端**：设置 → 播放器设置 →「音轨与字幕」分组 → **版本筛选（正则）**、**字幕筛选（正则）**、**音频筛选（正则）**。
- **TV 端**：设置 → 播放 → 对应的三个正则输入项。

三项各自独立，留空即「不启用」，回退到默认行为（首选字幕/音频语言、默认版本）。

## 它在匹配什么

正则不是匹配文件名，而是匹配每个候选「拼接出来的可匹配文本」：

| 设置项 | 匹配的文本来源 |
|---|---|
| 版本筛选 | 媒体源**名称** + **容器** + 视频**分辨率**（4K/1080p/720p…）+ 视频**编码** + 显示名 |
| 字幕筛选 | 字幕轨**显示名** + **标题** + **语言** + **编码** |
| 音频筛选 | 音频轨**显示名** + **标题** + **语言** + **编码** + **声道**（如 `6ch`） |

只要正则在这段文本里命中任意一处，就算匹配。

## 通用规则

- **大小写不敏感**，并开启 unicode —— 可以直接写中文（如 `简` `繁` `日`）。
- **留空 = 关闭**该筛选。
- 正则**非法不会保存**（会有提示），所以不用担心写错把选轨搞乱。
- 命中**优先级**：
  - 版本：详情页里手动选的版本 ＞ 正则命中 ＞ 列表第一个。
  - 字幕 / 音频：你在播放中手动切过的轨 ＞ 正则命中 ＞ 首选语言/服务端默认。
- 筛选只负责「**选哪一个**」，不改变播放方式 —— LinPlayer 只做流式直连、不转码。

## 版本筛选示例

```text
4K|2160            # 优先 4K / 2160p 片源
1080               # 优先 1080p
HEVC|265|x265      # 优先 HEVC/H.265 编码
原盘|remux|bluray   # 优先原盘 / Remux / 蓝光
HDR|DV|杜比视界     # 优先 HDR / 杜比视界
```

## 字幕筛选示例

例子里最常见的诉求是「要中文，简繁都行」：

```text
中文|简|繁|chi|zh|chs|cht   # 任意中文字幕（不分简繁）
简|chs|sc                  # 只要简体
繁|cht|tc|big5             # 只要繁体
双语|中.*日|中.*英          # 双语轨（中日 / 中英）
ass|ssa                    # 只要特效字幕轨
```

## 音频筛选示例

```text
jpn|日|日语                  # 日语音轨
国语|普通话|mandarin|cmn      # 国语
flac|truehd|atmos|7\.1       # 高规格音轨（注意 7\.1 里的点要转义）
eng|英                       # 英语
```

## 正则语法速查

| 写法 | 含义 | 例 |
|---|---|---|
| `a|b` | a 或 b | `chi|zh` |
| `.` | 任意一个字符 | `中.日` |
| `.*` | 任意长度任意字符 | `中.*日` |
| `\.` | 字面意义的点号 | `7\.1` |
| `( )` | 分组 | `(简|繁)体` |
| `[ ]` | 字符集合 | `[0-9]` |
| `^` `$` | 行首 / 行尾锚定 | `^4K$` |

> 小提示：`7.1` 里的 `.` 在正则里是「任意字符」，要精确匹配「7.1 声道」请写成 `7\.1`。

## 常见组合

- 想要「**4K 片源 + 中文字幕**」：版本筛选填 `4K|2160`，字幕筛选填 `中文|简|繁|chi|zh`，两项配合即可在进入多版本服务时自动选中 4K 并挂上中文字幕。
- 想要「**日语原声 + 简体中字**」：音频填 `jpn|日`，字幕填 `简|chs`。
